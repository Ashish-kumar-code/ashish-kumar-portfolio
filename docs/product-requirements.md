# Product Requirements Document (PRD)

## Project Name

Ashish Kumar Portfolio

---

# Vision

Create a premium portfolio website that showcases technical skills, engineering mindset, real-world projects, and continuous learning while maximizing recruiter engagement.

---

# Problem Statement

Most student portfolios suffer from one or more of the following:

- Generic templates
- Poor project presentation
- Weak storytelling
- Lack of engineering depth
- Slow loading speed
- Poor mobile responsiveness

This portfolio aims to solve these problems by combining modern UI, clear communication, and detailed technical case studies.

---

# Goals

Primary Goals

- Increase interview opportunities
- Demonstrate frontend skills
- Demonstrate backend understanding
- Showcase engineering thinking
- Highlight project impact

Secondary Goals

- Build personal brand
- Improve online presence
- Increase GitHub visibility
- Make resume easily accessible

---

# Success Metrics

- Lighthouse Score >95
- Mobile Friendly
- Responsive on all devices
- Fast loading
- Recruiters understand profile within 30 seconds
- Recruiters spend more time exploring projects than reading resume

---

# Non Goals

This website is NOT intended to be:

- A blog platform
- A social media website
- A dashboard application
- A portfolio template marketplace

---

# Target Roles

- Full Stack Developer
- Backend Developer
- Data Analyst
- AI/ML Engineer (Entry Level)

---

# Constraints

- Mobile First
- Accessible
- SEO Optimized
- Fast
- Easy to maintain
- JSON-driven content
